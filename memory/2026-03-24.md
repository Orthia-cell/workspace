## Root Cause Analysis: System Disruption

### What Happened
Multiple service interruptions occurred during Grace bot setup attempts.

### Primary Cause: Tailscale Serve Mode Failure
**Trigger:** Attempted to enable `gateway.tailscale.mode: serve` to allow Telegram servers to reach the gateway for Grace_Laere_bot.

**Result:** Gateway entered a failed/hung state:
- Restart commands failed or produced `missing tool result in session history` errors
- `gateway timeout after 60000ms` errors when querying status
- Gateway process unresponsive to SIGTERM, required SIGKILL

**Technical Details:**
- Original working state: `gateway.tailscale.mode: off`
- Failed state: `gateway.tailscale.mode: serve` + `gateway.bind: lan`
- The combination caused the gateway to hang during startup

### Contributing Factors
1. **Token Overwrite:** Grace's bot token replaced Orthia's, breaking both bots
2. **Workspace Confusion:** Attempted to switch workspaces, but OpenClaw only supports one global workspace
3. **Memory Loss:** User pressed "auto-fix" which cleared recent conversation context

### Recovery Actions Taken
1. Restored Orthia's original bot token from backup (`openclaw.json.bak.20260316113103`)
2. Disabled Tailscale mode: `openclaw config set gateway.tailscale.mode off`
3. Reset workspace to Orthia's: `/root/.openclaw/workspace`
4. Hard-killed gateway process and cold-started

### Current Status
- ✅ Orthia_bot restored and operational
- ⚠️ Grace_Laere_bot setup incomplete (requires separate solution)

### Lesson Learned
Tailscale serve mode + lan bind configuration causes gateway failure. Avoid this combination.
