Initial Bot Setup

Create a Bot with BotFather

  1. Open Telegram and search for @BotFather
  2. Start a chat and send /newbot
  3. Follow the prompts to name your bot and set a username (must end in "bot")
  4. Save the bot token BotFather gives you — you'll need it

Configure OpenClaw

Run these commands (replace YOUR_BOT_TOKEN with the actual token):

bash 
# Load nvm environment
source /root/.nvm/nvm.sh

# Set your bot token
openclaw config set channels.telegram.botToken "YOUR_BOT_TOKEN"

# Set DM policy to pairing (secure default)
openclaw config set channels.telegram.dmPolicy "pairing"

# Require mention in groups
openclaw config set channels.telegram.groups."*".requireMention true

# Enable the channel
openclaw config set channels.telegram.enabled true
💡 What These Settings Do
  • dmPolicy: "pairing" — Only paired users can DM the bot (for security)
  • requireMention: true — In groups, the bot only responds when mentioned

Restart the Gateway

bash 
openclaw gateway restart

Group Chat Settings

Basic Group Configuration

bash 
# Allow bot in all groups, require mention to respond
openclaw config set channels.telegram.groups."*".requireMention true

# Allow bot in a specific group only (replace with your group ID)
openclaw config set channels.telegram.groups."-1001234567890".requireMention true

# Allow bot to respond to ANY message in a specific group (no mention needed)
openclaw config set channels.telegram.groups."-1001234567890".requireMention false

# Set group policy: open, allowlist, or disabled
openclaw config set channels.telegram.groups."-1001234567890".groupPolicy "open"

Finding Your Group ID

  1. Add the bot to the group
  2. Send a message in the group
  3. Run: openclaw logs --follow
  4. Look for chat.id in the log output
💡 Quick Tip
Forward any group message to @userinfobot — it'll show the group ID instantly.

Privacy Mode (Important!)

Telegram bots default to Privacy Mode — they only see messages that mention them.

⚠️ To See All Group Messages
  1. Message @BotFather
  2. Send /setprivacy
  3. Select your bot
  4. Choose Disable
  5. Remove and re-add the bot to the group (required!)

Or just make the bot a group admin — that bypasses privacy mode too.

Custom Commands

Custom commands appear in Telegram's command menu (the / button). They don't auto-implement behavior — you handle them in your agent logic.

Adding Custom Commands

bash 
openclaw config set channels.telegram.customCommands '[{"command":"backup","description":"Git backup"},{"command":"generate","description":"Create an image"},{"command":"status","description":"Check system status"}]'

Command Rules

  • Names: lowercase, a-z, 0-9, _, 1-32 characters
  • Cannot override native commands like /new, /reset, /status
  • Duplicates are skipped

Handling Commands in Your Agent

When someone sends /backup, your agent receives it as regular text. You parse and handle it:

text 
User: /backup
Agent: Starting backup... [runs backup skill]

Native Commands (Built-in)

Command Description
/new or /reset Start fresh session
/status Show session status
/verbose on/off Toggle verbose mode
/think low/medium/high Set thinking level
/activation mention/always When to respond in groups
/compact Compress session history

Disable native commands:

bash 
openclaw config set channels.telegram.commands.native "disabled"

Pairing Flow

Pairing = explicit approval before someone can DM your bot. This is the default security model.

How It Works

  1. Stranger DMs your bot → Bot replies with a pairing code (8 characters)
  2. Message is NOT processed until approved
  3. You approve the code → They can now chat freely

Managing Pairing

bash 
# List pending pairing requests
openclaw pairing list telegram

# Approve a pairing code
openclaw pairing approve telegram ABC12345

# Reject/clear a pending request
openclaw pairing reject telegram ABC12345

Pairing Policies

bash 
# Require pairing for DMs (default, most secure)
openclaw config set channels.telegram.dmPolicy "pairing"

# Use allowlist instead (pre-approved user IDs)
openclaw config set channels.telegram.dmPolicy "allowlist"
openclaw config set channels.telegram.allowFrom '["123456789","987654321"]'

# Open to everyone (not recommended)
openclaw config set channels.telegram.dmPolicy "open"
openclaw config set channels.telegram.allowFrom '["*"]'

# Disable DMs entirely
openclaw config set channels.telegram.dmPolicy "disabled"

Finding Your Telegram User ID

Method 1 (safest):

  1. DM your bot
  2. Run: openclaw logs --follow
  3. Look for from.id in the logs

Method 2 (via API):

bash 
curl "https://api.telegram.org/bot<YOUR_BOT_TOKEN>/getUpdates"

Method 3 (third-party bot):

  • Message @userinfobot or @getidsbot

Quick Reference: Common Setups

🔒 Secure Setup (Recommended)

bash 
# DMs: pairing required
# Groups: mention required, allowlist for senders
openclaw config set channels.telegram.dmPolicy "pairing"
openclaw config set channels.telegram.groups."*".requireMention true
openclaw config set channels.telegram.groupPolicy "allowlist"

🌐 Open Group Bot

bash 
# Respond to all messages in a specific group
openclaw config set channels.telegram.groups."-1001234567890".requireMention false
openclaw config set channels.telegram.groups."-1001234567890".groupPolicy "open"

👤 Personal Assistant (Closed)

bash 
# Only you can DM, no groups
openclaw config set channels.telegram.dmPolicy "allowlist"
openclaw config set channels.telegram.allowFrom '["YOUR_USER_ID"]'
# Don't configure groups at all
